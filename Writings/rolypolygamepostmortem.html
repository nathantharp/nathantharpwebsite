Roly Poly Game Postmortem

The first game as part of my game a week challenge that I started after making TINMOM. I had the idea a few weeks ago after misreading role playing game as roly poly game. Not sure how that happened.

What went right

The main mechanic of rolling around and hitting enemies works very well. I find it quite fun to move around and bump into things. The movement feels like watching a pinball machine.

What went wrong.

After getting the main mechanic done, I wasn't sure what to do next with the game. So I had the idea of making a rogue-like. Big mistake as the scope of what I wanted to do with the rogue-like was way beyond what I could finish in a week. The result is a half-baked rogue-like with no ending. The features I added to make the roly poly game into a rogue-like I think actually hurts the main mechanic.

What I learned

Keep the scope of the game smaller. I also learned it is important to put more thought and planning behind major game decisions. I need to identify the core of the game and making sure additions to the game support that core idea. When deciding to make it a rogue-like, I did not think much about how that would affect the gameplay. The game probably would have been better with a small set of levels that put twists on the mechanic of rolling around. Instead of randomized levels.
